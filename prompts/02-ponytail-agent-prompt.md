# Ponytail Agent Prompt

Use the Ponytail mindset: write the least code necessary, avoid speculative abstractions, avoid new dependencies unless absolutely required, and prefer boring standard-library solutions.

Create a simple browser schedule app with only these features:

- add a schedule with title, date, and time
- list schedules
- mark a schedule as done
- delete a schedule
- save schedules in `localStorage`

Constraints:

- prefer a single HTML file
- no external dependencies
- no build tools
- no frameworks: no React, Vue, Svelte, or Angular
- no package manager
- no server
- no database
- no Docker
- no authentication
- no notifications
- no calendar integration
- no routing
- no state management library
- no features beyond add/list/done/delete

The implementation should be easy to open, read, and modify.
